================================================================================
GUÍA PARA AGREGAR NUEVAS PLANTILLAS DE TARJETA DIGITAL
================================================================================

Este documento describe el proceso completo para agregar una nueva plantilla
de tarjeta digital al sistema MuyLocal. Sigue estas instrucciones exactamente
para garantizar el correcto funcionamiento.

================================================================================
1. ARQUITECTURA DEL SISTEMA DE PLANTILLAS
================================================================================

El sistema utiliza:
- Vue 3 con Options API
- Estilos inline computados (NO CSS variables heredadas)
- Schema PHP para definir controles del editor
- LivePreview.vue como contenedor que renderiza la plantilla seleccionada

Flujo de datos:
  TemplateEditor.vue (editor visual)
       │
       ├── Carga schema desde API: GET /api/templates/{name}/schema
       ├── Carga settings desde API: GET /api/companies/{id}/settings
       │
       └── LivePreview.vue (contenedor)
               │
               └── Template[Nombre].vue (plantilla específica)
                   - Recibe prop: customization (objeto con valores)
                   - Usa computed properties para generar estilos inline
                   - Aplica estilos via :style="computedStyle"

4. ESTRUCTURA DEL COMPONENTE VUE
================================================================================

El componente debe seguir esta estructura exacta:

```vue
<template>
    <div class="card-template ejemplo" :style="containerStyle">
        <!-- Usar :style="computedStyle" en elementos dinámicos -->
        <header :style="headerStyle">
            <img v-if="company?.logo_path" :src="company.logo_path" :style="logoStyle">
        </header>

        <div class="avatar-container" v-if="showAvatar">
            <img v-if="card?.photo_path" :src="card.photo_path" :style="avatarStyle">
            <div v-else class="avatar-placeholder" :style="avatarStyle">
                <i class="bi bi-person"></i>
            </div>
        </div>

        <h1 :style="nameStyle">{{ card?.first_name }} {{ card?.last_name }}</h1>
        <p :style="jobTitleStyle">{{ card?.job_title }}</p>

        <!-- Botones sociales con método -->
        <div class="social-buttons">
            <a v-if="card?.whatsapp" :href="whatsappLink" :style="getSocialBtnStyle('whatsapp')">
                <i class="bi bi-whatsapp"></i>
            </a>
        </div>
    </div>
</template>

<script>
export default {
    name: 'TemplateEjemplo',

    props: {
        customization: { type: Object, default: () => ({}) },
        company: { type: Object, default: () => ({}) },
        card: { type: Object, default: () => ({}) },
        services: { type: Array, default: () => [] },
        products: { type: Array, default: () => [] }
    },

    computed: {
        // Link de WhatsApp (patrón común)
        whatsappLink() {
            if (!this.card?.whatsapp) return '#'
            const phone = this.card.whatsapp.replace(/\D/g, '')
            return `https://api.whatsapp.com/send?phone=${phone}`
        },

        // Visibilidad de avatar
        showAvatar() {
            return this.customization?.avatar?.mostrar !== false
        },

        // Estilo del contenedor principal
        containerStyle() {
            const gen = this.customization?.general || {}
            return {
                backgroundColor: gen.colorFondo || '#ffffff',
                fontFamily: `'${gen.tipografia || 'Inter'}', sans-serif`,
                color: gen.colorTexto || '#333333',
            }
        },

        // Estilo del avatar (IMPORTANTE: width === height para círculo)
        avatarStyle() {
            const avatar = this.customization?.avatar || {}
            const tamano = avatar.tamano ?? 120  // ?? para numéricos
            return {
                width: `${tamano}px`,
                height: `${tamano}px`,
                borderRadius: `${avatar.radio ?? 50}%`,
                objectFit: 'cover',
            }
        },

        // Estilo del nombre
        nameStyle() {
            const gen = this.customization?.general || {}
            return {
                fontSize: '1.5rem',
                fontWeight: '600',
                color: gen.colorTexto || '#333333',  // || para strings
            }
        },

        // Más computed styles...
    },

    methods: {
        // Método para estilos paramétricos (botones)
        getSocialBtnStyle(type) {
            const config = this.customization?.botones || {}
            const estilo = config.estilo || 'solido'

            const base = {
                borderRadius: `${config.radio ?? 50}%`,
                transition: 'all 0.3s ease',
            }

            if (estilo === 'solido') {
                return {
                    ...base,
                    backgroundColor: config.colorBoton || '#333333',
                    color: config.colorTextoBoton || '#ffffff',
                }
            } else if (estilo === 'borde') {
                return {
                    ...base,
                    backgroundColor: 'transparent',
                    border: `1px solid ${config.colorBoton || '#333333'}`,
                    color: config.colorBoton || '#333333',
                }
            }
            // fantasma/minimalista
            return {
                ...base,
                backgroundColor: 'transparent',
                color: config.colorBoton || '#333333',
            }
        },
    }
}
</script>

<style scoped>

.card-template.ejemplo {
```

5. REGLAS CRÍTICAS PARA ESTILOS
================================================================================

✅ CORRECTO - Usar estilos inline computados:
```vue
<div :style="avatarStyle">

computed: {
    avatarStyle() {
        return {
            width: `${this.customization?.avatar?.tamano ?? 120}px`,
            borderRadius: `${this.customization?.avatar?.radio ?? 50}%`,
        }
    }
}
```

❌ INCORRECTO - NO usar CSS variables heredadas:
```css
/* ESTO NO FUNCIONA con CSS scoped */
.avatar {
    width: var(--avatar-tamano, 120px);
}
```

❌ INCORRECTO - NO usar calc() con CSS variables:
```css
/* ESTO NO FUNCIONA */
.avatar {
    width: calc(var(--avatar-tamano) * 1px);
}
```

OPERADORES PARA DEFAULTS:
- Usar ?? para valores numéricos: avatar.tamano ?? 120
- Usar || para strings/colores: gen.colorTexto || '#333333'

```javascript

    TemplateModern,
    TemplateClassic,
    TemplateMinimal,
    TemplateCreative,
    TemplateCyber,
    TemplateVibrant,
    TemplateEjemplo,  // <-- AGREGAR

```

3. Agregar al mapeo en currentTemplateComponent:
```javascript
currentTemplateComponent() {
    const templates = {
        modern: 'TemplateModern',
        classic: 'TemplateClassic',
```

13. PATRONES COMUNES
================================================================================

LINK DE WHATSAPP:
```javascript
whatsappLink() {
    if (!this.card?.whatsapp) return '#'
    const phone = this.card.whatsapp.replace(/\D/g, '')
    return `https://api.whatsapp.com/send?phone=${phone}`
```

CLASES DINÁMICAS (para animaciones):
```javascript
// En computed
avatarAnimationClass() {
    const anim = this.customization?.avatar?.animacion || 'ninguna'
    if (anim === 'flotar') return 'anim-float'
    if (anim === 'latir') return 'anim-pulse'
    return ''
}

// En template
<div :class="avatarAnimationClass">

// En CSS
.anim-float {
    animation: float 4s ease-in-out infinite;
}
@keyframes float {
    0%, 100% { transform: translateY(0); }
    50% { transform: translateY(-10px); }
}
```

DEGRADADOS DINÁMICOS:
```javascript
backgroundStyle() {
    const gen = this.customization?.general || {}
    const color1 = gen.color1 || '#ff00cc'
    const color2 = gen.color2 || '#333399'
    return {
        backgroundImage: `linear-gradient(-45deg, ${color1}, ${color2})`,
    }
}
```

BOTONES CON MÚLTIPLES ESTILOS:
```javascript
getSocialBtnStyle(type) {
    const config = this.customization?.botones || {}
    const estilo = config.estilo || 'solido'
    const base = { borderRadius: `${config.radio ?? 50}%` }

    if (estilo === 'solido') {
        return { ...base, backgroundColor: config.color, color: '#fff' }
    } else if (estilo === 'borde') {
        return { ...base, backgroundColor: 'transparent', border: `1px solid ${config.color}` }
    }
    return { ...base, backgroundColor: 'transparent', color: config.color }
}
```

15. ERRORES COMUNES Y SOLUCIONES
================================================================================

ERROR: Los controles del editor no afectan la plantilla
CAUSA: Usando CSS variables en lugar de estilos inline computados
SOLUCIÓN: Cambiar de var(--nombre) a :style="computedProperty"

ERROR: Foto aparece como óvalo en lugar de círculo
CAUSA: width y height diferentes
SOLUCIÓN: Asegurar width === height en el computed style

ERROR: Fuente no cambia al seleccionar otra tipografía
CAUSA: Fuente no está cargada en Google Fonts
SOLUCIÓN: Agregar la fuente al link en app.blade.php

ERROR: Valores numéricos no funcionan (ej: tamaño en px)
CAUSA: Falta concatenar la unidad en el computed
SOLUCIÓN: Usar template literals: `${valor}px`

ERROR: Cambios en schema no se reflejan
CAUSA: Caché de Laravel no limpiada
SOLUCIÓN: Ejecutar php artisan config:clear && cache:clear

ERROR: undefined cuando accedo a customization
CAUSA: No usar optional chaining
SOLUCIÓN: Usar this.customization?.seccion?.campo
